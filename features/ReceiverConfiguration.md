# Receiver Configuration

## Objective
Allow users to define how they can be contacted.

## Scope
- Category enable/disable
- Required fields per category
- Request caps
- Contact reveal method

## Behavior
- Changes apply immediately
- Email proxy behavior depends on configuration

## Constraints
- Simple UX
- No nested configurations

## Deliverables
- Settings UI
- Persistence logic

## Acceptance Criteria
- Users can control inbound volume
- Email is disabled when required fields exist
- Misconfigurations are prevented
